# index-m-hub

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于存放各类独立开发的 HTML 页面资源，包括但不限于静态展示页、小工具、交互原型等。每个页面均为独立的文件，不依赖特定域名或后端服务，适合快速部署与分享。

## 目录说明

```
index-m-hub/
├── pages/        # 存放所有独立 HTML 页面
│   ├── page-a/   # 示例子目录（按需组织）
│   ├── page-b/
│   └── ...
├── assets/       # 公共资源文件（CSS、JS、图片等）
└── README.md     # 本文件
```

- **pages/**：每个子目录对应一个独立页面，包含其 HTML 文件及私有资源。
- **assets/**：多个页面可复用的公共资源，如通用样式、脚本库等。

## 页面归档说明

- 每个页面应保持独立、自包含，尽量避免跨目录引用。
- 页面文件命名建议简洁明了，便于识别内容。
- 如需引用公共资源，请使用相对路径，确保部署后仍可正常访问。

## 维护说明

- 新增页面：在 `pages/` 下创建新目录，放置 HTML 及相关文件；如需共享资源，可更新 `assets/`。
- 更新页面：直接修改对应文件，注意保持向后兼容或标注版本。
- 删除页面：移除对应目录及所有相关引用，避免残留文件。
- 欢迎提交 Pull Request 贡献新页面或改进现有内容。
